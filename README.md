## рҹ“Ң DescripciГіn

Desarrollado como proyecto acadГ©mico para la materia **IngenierГӯa de Software II** вҖ” Universidad PedagГіgica y TecnolГіgica de Colombia (UPTC).

## вҡҷпёҸ Microservicios

| # | Microservicio | Lenguaje / Framework | Base de Datos | Puerto |
|---|---|---|---|---|
| 1 | **ms-autenticacion** | Node.js + Express | PostgreSQL | 8081 |
| 2 | **ms-usuarios** | Node.js + Express | PostgreSQL | 8082 |
| 3 | **ms-citas** | Python + FastAPI | MongoDB | 8083 |
| 4 | **ms-pacientes** | Python + FastAPI | MongoDB | 8084 |
| 5 | **ms-historial** | Java + Spring Boot | MongoDB | 8085 |

### ms-autenticacion
Gestiona el acceso al sistema mediante autenticaciГіn con **JWT (JSON Web Tokens)**. Implementa el patrГіn arquitectГіnico *Access Token*.

- `POST /auth/login` вҖ” Iniciar sesiГіn
- `POST /auth/logout` вҖ” Cerrar sesiГіn
- `POST /auth/validate` вҖ” Validar token

### ms-usuarios
AdministraciГіn del personal del sistema (optГіmetras y asistentes).

- `POST /usuarios` вҖ” Registrar usuario
- `GET /usuarios/{id}` вҖ” Consultar usuario
- `PUT /usuarios/{id}` вҖ” Modificar usuario
- `PATCH /usuarios/{id}/inhabilitar` вҖ” Inhabilitar usuario

### ms-citas
GestiГіn del agendamiento de citas mГ©dicas.

- `POST /citas` вҖ” Crear cita
- `GET /citas/{id}` вҖ” Consultar cita
- `PUT /citas/{id}` вҖ” Modificar cita
- `DELETE /citas/{id}` вҖ” Cancelar cita
- `PATCH /citas/{id}/confirmar` вҖ” Confirmar cita

### ms-pacientes
Registro y administraciГіn de la informaciГіn de pacientes.

- `POST /pacientes` вҖ” Registrar paciente
- `GET /pacientes/{id}` вҖ” Consultar paciente
- `PUT /pacientes/{id}` вҖ” Modificar paciente
- `DELETE /pacientes/{id}` вҖ” Eliminar paciente

### ms-historial
GestiГіn del historial clГӯnico y documentaciГіn de exГЎmenes visuales completos.

- `POST /historial` вҖ” Crear historia clГӯnica
- `GET /historial/{id}` вҖ” Consultar historial
- `POST /historial/{id}/examen` вҖ” Documentar examen visual
- `GET /historial/{id}/examen/{examenId}` вҖ” Consultar examen
- `GET /historial/{id}/documento` вҖ” Generar documento HC
- `GET /historial/{id}/rx` вҖ” Generar Rx final

## рҹҡҖ InstalaciГіn y ConfiguraciГіn

### 1. Clonar el repositorio

```bash
git clone https://github.com/tu-usuario/optica.git
cd optica
```

### 2. Levantar la infraestructura (bases de datos + Eureka)

```bash
docker compose up --build
```

Esto levanta automГЎticamente:
- Spring Eureka en `http://localhost:8761`
- PostgreSQL en el puerto `5432`
- MongoDB (instancia 1) en el puerto `27017`
- MongoDB (instancia 2) en el puerto `27018`

### 3. Verificar que Eureka estГЎ corriendo

Abre `http://localhost:8761` en el navegador. DeberГӯas ver el dashboard de Eureka.

### 4. Levantar cada microservicio

Cada microservicio tiene su propio `README.md` con instrucciones de configuraciГіn. En general:

```bash
# ms-autenticacion y ms-usuarios (Node.js)
cd ms-autenticacion
npm install
npm run dev

# ms-citas y ms-pacientes (Python)
cd ms-citas
pip install -r requirements.txt
uvicorn main:app --reload --port 8083

# ms-historial (Java)
cd ms-historial
./mvnw spring-boot:run
```
